## 2024-07-02 - Missing State Validation in GitHub App OAuth Flow
**Vulnerability:** A CSRF/IDOR vulnerability in the GitHub App installation flow where an attacker could potentially link their own GitHub App installation to a victim's user account, because the `/api/github/setup` endpoint lacked a `state` parameter validation check.
**Learning:** This existed because the frontend directly linked to the GitHub App installation URL (`https://github.com/apps/.../installations/new`) without generating or tracking an OAuth state parameter. The Next.js API route that handles the redirect back from GitHub blindly trusted the provided `installation_id`.
**Prevention:** Always use a state parameter (like `crypto.randomBytes(32).toString('hex')`) stored securely in an `HttpOnly` cookie before redirecting a user to an external OAuth-like setup flow. Verify that the returned state matches the cookie state before associating the returned external IDs with the currently logged-in user.
